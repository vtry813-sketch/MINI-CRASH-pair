Architecture:

  • Server: Node.js / Express
  • Storage: Redis for sessions
  • Queue: BullMQ
  • Frontend: Static HTML/CSS + minimal JS

Design notes: Add gradients, neon borders, glass morphism, and subtle particles for modern aesthetic.

Accessibility tips: Provide alt text, focus states, keyboard navigation, and sufficient contrast.

Deployment tips: Use Railway / Vercel / Render for quick deploy, and activate automatic restarts.

Security: Keep secrets out of repo. Use environment variables and secret managers.

Monitoring: Integrate Sentry or Logflare for logs & error tracking.

Testing: Unit tests + e2e tests (Playwright) recommended for production bots.

Performance: Serve assets with CDN and compress responses (brotli/gzip).
